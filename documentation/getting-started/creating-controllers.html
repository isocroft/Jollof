  • ยง Choose Controller Name

    To get started on choosing a controller name, you have to know that controller names are linked with route names. Whenever you choose a name, choose one that has relevance to the function or group of functions it (Controller) will perform as far as your application is concerned.

    For instance, for the user authorization/authentication part of your application, you could choose Accounts as the name for controller. As a result, you can have /accounts/login, /accounts/reset-password/@user and /accounts/logout as possible routes. In the routes, the parts that have an @ symbol in front are route parameters.

    All controller classes must extend the core Controller class.

  • ยง Create Controller File

    Every controller class file can be created in 2 ways:

    1. Using the console command for creating the controller class file.
    2. Manually creating the controller class file and registering it in the system/base/class_maps.php file.

    When the file has been created, navigate to the controllers folder and open the file in a text editor. You should see something like as shown below:

    Showcase: 
    class Accounts extends Controller {

    protected $params = array();

    public function __construct(array $params = array()){

        parent::__construct($params);

    }

    // @Override

    public function index($models){

        # code ...

    }
}

  • ยง Controller Method Argument

    Only one argument is passed to every method of any controller. This is an array of Models (objects) that are defined for the given route for which the controller method is invoked.
